# denkr 0.3.0

## Summary

denkr 0.3.0 is a broader beta update focused on making the assistant more reliable in daily use. This release improves provider support, longer chat continuity, automatic backups, memory indexing, localization foundations, and several chat interaction details that affect the feel of the app.

## What changed

### Assistant continuity and memory

- Added continuity compaction for longer conversations, so older chat context can be summarized and carried forward without replaying the entire transcript every turn.
- Improved session summary behavior so the assistant has a clearer compact record of what happened in previous parts of the conversation.
- Added automatic memory indexing batches, which helps saved memory become available to retrieval without relying on manual one-off indexing work.
- Added more runtime checks and tests around context budgets, session segments, memory worker preferences, and continuity behavior.

### Providers and models

- Added Gemini provider support and continued stabilizing the Gemini runtime after the first integration.
- Rebuilt the Gemini Google account connection flow, including compatibility work for older OAuth functions and a more stable account runtime.
- Added NVIDIA NIM provider support as a new model platform option.
- Added Grok 4.3 model support across the provider catalog, runtime configuration, and related tests.
- Improved provider fallback behavior and connection testing, so failed or unavailable providers can be handled more cleanly.

### Backups and beta updates

- Added daily automatic backups to reduce the chance of losing local app data during beta testing.
- Hardened backup export behavior and improved the backup settings surface.
- Improved beta versioning rules and update handling for Android beta installs.
- Fixed package lock syncing during debug version bumps so version changes stay consistent across release-related files.

### Chat and composer polish

- Fixed chat auto-scroll while responses are streaming, especially near the bottom of the conversation.
- Improved keyboard-safe chat layout so messages and composer controls behave better when the keyboard is open.
- Refined chat composer menu layering so menus appear in the right visual order and do not fight with nearby UI.
- Fixed bold markdown formatting inside messages.
- Improved chat bubble markdown and transcript persistence behavior.
- Fixed assistant sidebar tone text truncation so longer tone descriptions display more cleanly.

### Tools and web actions

- Added a YouTube video search tool action so the assistant can search for videos as part of tool-driven work.
- Improved YouTube transcript handling and related web tool behavior.
- Added tests and documentation around web search and YouTube tool behavior.

### Localization foundation

- Introduced the app localization system with English, German, and Arabic locale files.

## How to use it

- Update from the Android beta channel once the 0.3.0 APK is published.
- Try longer chats and check whether the assistant keeps context more naturally after the conversation grows.
- Check provider settings if you use Gemini, Grok, or NVIDIA NIM, because this release changes provider catalog and connection behavior.
- Check backup settings and confirm daily automatic backups match your testing expectations.
- Try normal chat flows with the keyboard open, streaming responses, markdown formatting, and composer menus.
- If testing localization, switch through English, German, and Arabic surfaces and report any untranslated or awkward text.

## Known issues

- Android beta updates are still manual installs outside Google Play.
- The APK install confirmation still happens through the Android system installer.
- iOS is still not wired into this public Android beta release channel.
- Localization is newly introduced and should be treated as a foundation pass, not final copy quality in every language.
- Provider availability can still depend on account access, model access, regional restrictions, and upstream provider behavior.
